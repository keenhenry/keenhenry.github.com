Things I Want To Learn

27 Dec 2020
Tweet

Here I list a few things I want to learn recently so that I can keep track of my progress:

  • Improve your Bash scripting skill by writing some scripts to automate some of your routines
  • Learn how to use cron and crontab.
  • Learn to use tmux to improve your work flow.
  • Learn how to use vim cscope.
  • Learn some geometry simplification algorithms.
  • Study ‘longest common subsequence problem’ by reading ‘An Algorithm for Differential File Comparison’ paper.
  • Improve your Linux system administrative skills.
  • Learn to write a simple crawler and use it to crawl your own website to find dangling links.
  • Learn how to write Vimscript.

Books I wish to complete reading this year (2013):

  • C++ Hackers Guide
  • Python Geospatial Development
  • Great Modern Buildings

The ultimate learning goal for this year (2013):

  • Improve your C++ domain knowledge
  • Improve your bash scripting capabilities
  • Improve your Python domain knowledge
  • Learn some more algorithms, any algorithms; but the ones related to work first
  • Improve your Linux administration skills
  • Improve your UNIX-like OS domain knowledge; this include C programming language and maybe network programming
  • Improve your database domain knowledge, especially geospatial database
  • Improve your workflow by mastering some more tools (vim, tmux, bash, cron)

Time management plan (2013):

  • Do studying, reading, learning work during weekdays.
  • Do notes taking, ideas written down during weekdays.
  • Write small scripts and small programs also during weekdays, if you feel capable.
  • Do more extensive software development, or more complicated tasks, during weekends, if any.
  • Do more extensive readings (including ‘extra-curriculum books’) also during weekend.
  • Study Dutch during weekends.

P.S. This list will grow and shrink over time.
